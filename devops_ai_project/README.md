# 📌 Project Overview

This project is my **first AI Agent project** built using **CrewAI** and **Ollama**.

The objective of this project is to understand how **multiple AI agents collaborate** to solve a real-world DevOps problem.

Instead of using a single AI model, two specialized AI agents work together to research and document the latest Kubernetes updates.

---

# 🎯 Project Objective

The primary objective of this project is to build an AI-powered DevOps assistant that can:

- Research the latest Kubernetes releases
- Collect new features and enhancements
- Identify deprecated APIs
- Gather security updates
- Summarize release notes
- Generate a structured Markdown report

This project demonstrates how multiple AI agents can collaborate to automate technical research and documentation.

Workflow

```
Sequential
```

Agents

```
2
```

Tasks

```
2
```

Verbose Logging

```
Enabled
```

Initially

```
Memory = true
Planning = true
```

Later changed to

```
Memory = false
Planning = false
```

Reason

CrewAI attempted to use OpenAI planning, which required an OpenAI API key. Since this project uses a local Ollama model, planning and memory were disabled.

# ⚠️ Challenges Faced

| Issue | Solution |
|-------|----------|
| Python 3.10 compatibility issue | Upgraded to Python 3.12 |
| `onnxruntime` installation error | Created a new Python 3.12 environment |
| Missing `pip` in virtual environment | Recreated the venv using `uv venv --seed` |
| `OPENAI_API_KEY` required | Disabled CrewAI planning and memory for Ollama-only setup |
| Tool execution failures | Verified the multi-agent workflow first; tool integration will be improved later |
